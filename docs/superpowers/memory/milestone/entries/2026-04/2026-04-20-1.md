## Title
建立基于标签的 macOS 命令行发布流程

## Why This Direction
项目已经具备稳定的本地测试能力，但还没有统一的远程校验与发布入口。为避免手工发版步骤分散、产物不一致，本次选择由 GitHub Actions 在固定事件上接管测试与发布。

## Direction
仓库后续通过两条标准流水线推进交付：拉取请求与标签推送触发自动测试；`v*` 标签推送额外触发 macOS 可执行文件构建，并上传为 GitHub Release 附件。

## Boundary
本次只覆盖 GitHub Actions 校验与发布工作流，以及 README 中对应说明。不包含签名、安装包封装、跨平台发布和版本管理策略扩展。

## Lasting Impact
- 后续正式发版以 `v*` 标签作为统一入口，而不是依赖本地手工上传。
- 发布产物保持为 macOS 命令行二进制压缩包，避免为当前工具引入额外安装包形态。
