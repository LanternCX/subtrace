# Subroute AS 拓扑报告实现计划

## 目标

将当前报告切换为以 AS 拓扑为主的阅读方式：删除单独的 IP 归属地查询，保留稳定的 AS 查询结果，并在报告开头输出按层汇总的拓扑总表。

## 任务拆分

### 任务 1：收缩标注模型

- 删除单独的 IP 归属地查询链路。
- 将地区字段改为 AS 查询结果中的国家或区域信息。
- 保留 IP、AS 号、组织、备注等最小数据模型。

### 任务 2：实现 AS 拓扑汇总

- 从每个节点的去程路线中提取有效 IP。
- 按层和 AS 进行聚合。
- 为每个聚合项汇总经过该层该 AS 的节点名称列表。

### 任务 3：重构报告布局

- 在文档开头新增拓扑总表。
- 将节点明细表改为 `IP / 地区 / AS 号 / 组织` 四列。
- 保持节点按国家标题分组，并在组内按名称排序。

### 任务 4：更新异常展示

- 删除已经无效的归属地查询失败提示。
- 保留解析失败、Ping 失败、Traceroute 失败和 AS 查询失败等仍有价值的异常信息。

### 任务 5：补齐验证

- 为拓扑总表、节点明细表和 AS 地区解析补充自动化测试。
- 运行测试、格式化和构建。
- 用真实订阅进行一次实跑验证。
