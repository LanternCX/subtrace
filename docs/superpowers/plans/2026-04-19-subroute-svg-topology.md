# Subroute SVG 拓扑总览实现计划

## 目标

将当前 Markdown 报告开头的 AS 拓扑总表替换为 SVG 树状图，并在 Markdown 中直接引用该 SVG 文件。

## 任务拆分

### 任务 1：建立 SVG 拓扑模型

- 从现有路由数据中提取 AS 路径。
- 将共享的 AS 节点按层合并。
- 为机场节点建立叶子节点。
- 保留真实上下游连线关系。

### 任务 2：生成 SVG 文件

- 使用从左到右的树状布局。
- 中间节点显示 AS 号、地区、组织。
- 叶子节点显示机场节点名。
- 没有可绘制路径时输出清晰的空态图。

### 任务 3：接入 Markdown 报告

- 在 Markdown 摘要后引用生成出的 SVG 文件。
- 删除原先的拓扑总表。
- 保留现有节点分组和节点明细表。

### 任务 4：补齐验证

- 为 SVG 结构和 Markdown 引用补充自动化测试。
- 运行测试、格式化和构建。
- 用真实订阅生成一次报告和 SVG，确认两者能配套查看。
