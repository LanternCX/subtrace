# Subroute 横向关系图设计说明

## 目标

将当前 HTML 页面中的图形主视图调整为横向关系图，而不是伪装成树；同一个 AS 在图中只保留一个节点，末端机场节点按最终 AS 聚合为可展开的组节点。

## 范围

- 保持 HTML 作为最终主产物。
- 图形主视图改为横向关系图。
- 同一个 AS 全图只保留一个节点。
- 末端机场节点不直接全量展开。
- 末端节点按最终 AS 聚合为组节点，按需展开。

## 图模型

- 关系图按从左到右排列。
- 图不再伪装成严格树结构。
- 同一个 AS 节点在全图中只出现一次。
- 允许出现共享节点和交叉连线。

## 末端聚合

- 机场节点不直接全部挂在主图上。
- 主图末端先显示“按最终 AS 聚合”的组节点。
- 同一个最终 AS 下的机场节点合并到同一个组节点中。
- 点击组节点后，再在页面中展开该组里的具体机场节点。

## 页面交互

- 图整体为横向布局。
- 点击 AS 节点：展示该 AS 的详情和关联范围。
- 点击末端组节点：展示该组包含的机场节点列表。
- 点击具体机场节点：展示该节点的详细路由信息。

## 页面结构

### 顶部

- 订阅地址
- 概览信息
- 搜索与筛选入口

### 中部

- 横向关系图
- 图支持缩放与拖拽浏览

### 底部

- 当前选中对象的详情
- 地区分组与补充列表

## 成功标准

- 图主视图改为横向关系图。
- 同一个 AS 在图中只出现一次。
- 末端机场节点按最终 AS 聚合显示。
- 点击聚合节点后可以查看组内具体机场节点。
- 点击具体机场节点后仍能查看路由明细。
