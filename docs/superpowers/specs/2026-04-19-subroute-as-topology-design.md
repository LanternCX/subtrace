# Subroute AS 拓扑报告设计说明

## 目标

将当前报告调整为以 AS 路径为主的阅读方式：不再查询单独的 IP 归属地，只保留稳定的 AS 查询结果，并在报告开头提供按层汇总的路由拓扑总表。

## 范围

- 继续支持 `subroute [url]` 直接执行。
- 输入仍为 Base64 订阅地址。
- 节点测试范围仍为本机到节点入口地址的去程路线。
- 节点探测仍保留基础连通性检查和去程路由探测。
- 报告不再包含单独的 IP 地理归属查询结果。
- 报告中的“地区”统一改为 AS 查询结果自带的国家或区域标识。

## 数据来源

- IP 到 AS 号的映射继续使用现有的 AS 查询方式。
- AS 组织名称继续使用现有的 AS 查询方式。
- 地区字段直接取 AS 查询结果中自带的国家或区域标识。
- 不再对每个 IP 单独发起地理归属查询。

## 报告结构

### 顶部摘要

- 订阅地址
- 生成时间
- 总条目数
- 可测节点数
- 无法解析条目数

### 路由拓扑总表

放在文档开头摘要之后，字段固定为：

- 层
- 地区
- AS 号
- 组织
- 节点

汇总规则：

- 以“层 + AS”作为一行的聚合单位。
- 同一层中命中同一个 AS 的多个节点，合并到同一行。
- 节点列展示经过该层该 AS 的机场节点名列表，去重后按名称排序。
- 地区、AS 号、组织缺失时统一写为“未知”。

### 节点分组

节点主体部分继续按国家标题分组，顺序固定为：

1. 美国
2. 日本
3. 香港
4. 新加坡
5. 台湾
6. 未分类

分组口径：

- 根据节点名称命中常用关键词做归类。
- 常用别名一并纳入，例如狮城归到新加坡。
- 未命中上述规则的节点归到未分类。

### 节点小节

每个节点仍保留基础摘要信息：

- 节点名
- 协议
- 入口地址
- 端口
- Ping 摘要

每个节点的路由表改为简化结构，字段固定为：

- IP
- 地区
- AS 号
- 组织

节点表规则：

- 不再显示层字段。
- 不再显示原始跳数字符串。
- 不再显示单独的城市归属结果。
- 仅展示去程路线中能够提取出的有效 IP 行。
- 同一节点内按路线出现顺序展示。
- 同一跳中重复出现的相同 IP 只保留一次。
- AS 信息缺失时，相关列统一写为“未知”。

## 异常处理

- `ping` 失败不影响节点进入报告。
- `traceroute` 中途超时不影响已拿到的 AS 路径展示。
- 单个 IP 查不到 AS 时，只将该行标记为“未知”，不阻塞其他行。
- 单个节点探测失败不阻塞整份报告生成。

## 成功标准

- 报告开头包含按层汇总的 AS 拓扑总表。
- 总表能够按层展示该层出现的 AS 和经过该层的节点列表。
- 每个节点的路由表仅保留 `IP / 地区 / AS 号 / 组织` 四列。
- 报告中不再出现单独的 IP 地理归属查询内容。
- 节点主体部分按固定国家标题顺序组织，并在组内按名称排序。
