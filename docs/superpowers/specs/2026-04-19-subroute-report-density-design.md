# Subroute 报告页信息密度调整设计说明

## 目标

在不改变核心交互的前提下，提高 HTML 报告页的信息密度：缩小关系图展示体量，将详细信息固定在右侧较窄区域，并删除与顶部筛选重复的地区分组区块。

## 范围

- 保留单一“关系图”主视图，不再引入第二个并列标签页。
- 保留顶部概览、搜索、地区筛选和缩放操作。
- 将关系图与详细信息改为同一行并排展示。
- 删除底部“地区分组”区块及其对应页面内容。
- 窄屏下继续允许上下堆叠，避免页面不可用。

## 页面结构

### 顶部控制区

- 保留订阅地址与概览卡片。
- 保留搜索、缩放、重置筛选。
- 保留地区筛选，作为唯一的地区维度入口。

### 主工作区

- 左侧为关系图区域，仍然是页面主视图。
- 右侧为详细信息区域，宽度明显小于关系图区域。
- 两个区域放在同一个并排工作区中，视觉上强调“左侧浏览、右侧补充”。

## 视觉调整

- 缩小关系图容器的留白与默认视觉占比。
- 缩小详细信息卡片间距和局部留白，提高单位面积内的信息量。
- 保持现有整体风格，不引入新的视觉层级或新的交互入口。

## 交互影响

- 点击关系图节点后，右侧继续展示对应对象的详细信息。
- 顶部搜索和地区筛选继续作用于关系图与详情中的可点击对象。
- 删除地区分组后，不再提供底部按地区展开的第二套导航入口。

## 成功标准

- 页面只保留一个关系图主视图。
- 关系图与详细信息在宽屏下左右并排显示。
- 详细信息区域宽度小于关系图区。
- 底部“地区分组”区块完全移除。
- 地区筛选能力仍然保留且可正常工作。
