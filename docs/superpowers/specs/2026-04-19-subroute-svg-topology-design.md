# Subroute SVG 拓扑总览设计说明

## 目标

将当前报告开头的 AS 拓扑总表替换为 SVG 树状图，并在 Markdown 中直接引用该 SVG 文件，使总览层更直观，而节点明细部分继续保留文字表格。

## 范围

- 保留当前的节点测试方式与节点明细表。
- 仅替换文档开头的总览部分。
- 生成独立的 `.svg` 文件。
- Markdown 报告中直接引用该 `.svg` 文件。
- SVG 使用从左到右的树状布局。

## 总览图结构

- 起点在最左侧，表示去程路线的近端。
- 路径按层逐步向右展开。
- 中间节点表示 AS。
- 叶子节点表示机场节点名。

## 图节点内容

### AS 节点

每个 AS 节点显示：

- AS 号
- 地区
- 组织

说明：

- 地区继续使用 AS 查询自带的国家或区域结果。
- 组织使用 AS 查询结果。
- 地区或组织缺失时统一写为“未知”。

### 叶子节点

每个叶子节点直接显示机场节点名。

说明：

- 不再额外显示“命中节点数”。
- 树本身已经承载了路径和节点归属信息。

## 聚合规则

- 同一层中相同 AS 共享同一个图节点。
- 相邻两层之间按真实出现过的上下游关系连线。
- 某条路线走到最后一个可识别 AS 后，再连接到对应机场节点叶子。
- 叶子节点允许多个不同 AS 汇入，但同一机场节点名只保留一个叶子。

## Markdown 结构

- 顶部摘要保持不变。
- 摘要后新增 SVG 引用。
- SVG 下方不再保留原先的拓扑总表。
- 后续节点分组和节点明细表保持现有结构。

## 文件输出

- Markdown 继续输出到原报告文件。
- SVG 文件与 Markdown 文件放在同一目录。
- 文件名使用同一前缀，便于配套查看。

## 成功标准

- 执行后同时生成 Markdown 报告和对应的 SVG 总览图。
- Markdown 能直接引用生成出的 SVG。
- SVG 能以从左到右的树状结构展示 AS 路径与叶子节点。
- 中间节点显示 `AS号 / 地区 / 组织`。
- 叶子节点显示机场节点名。
- 节点明细部分继续保留文字表格，不被 SVG 替代。
