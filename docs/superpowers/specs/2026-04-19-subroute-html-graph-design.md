# Subroute HTML 关系图设计说明

## 目标

将当前基于 Markdown 和 SVG 的结果替换为单一的 HTML 主页面，使最终结果以可交互的关系图形式呈现，并在同一页面内查看节点明细。

## 范围

- 最终主产物改为一个独立的 HTML 文件。
- 不再以 Markdown 作为主入口。
- 页面整体采用从上到下的布局。
- 中间主体区域使用关系图展示 AS 与机场节点的整体关系。
- 页面下方展示当前选中对象的详细信息。

## 图模型

- 同一个 AS 号在全图中只保留一个节点。
- 图不强制保持树结构。
- 图允许出现真实关系带来的交叉和回连。
- 机场节点作为终点节点加入图中。

## 页面结构

### 顶部区域

- 报告基础信息
- 数据概览
- 筛选入口

### 中间区域

- 主关系图
- 关系图承担默认焦点
- 默认直接展示完整图谱

### 底部区域

- 当前选中对象的详细信息
- 点击 AS 节点后展示该 AS 的相关信息
- 点击机场节点后展示该节点的详细路由表

## 节点类型

### AS 节点

每个 AS 节点展示：

- AS 号
- 地区
- 组织

规则：

- 地区继续使用 AS 查询自带的国家或区域结果。
- 地区或组织缺失时统一写为“未知”。

### 机场节点

每个机场节点展示机场节点名。

## 交互原则

- 不做 fold / unfold。
- 通过点击、筛选和高亮进行浏览。
- 点击任意节点时，在底部详情区展示对应信息。
- 筛选负责收窄范围、定位和高亮，不负责修改图结构。

## 分组与筛选

- 支持按地区进行粗分组筛选。
- 分组可按字典序展示。
- 筛选既可作用于 AS，也可作用于机场节点。
- 默认先展示完整图谱，不预设折叠状态。

## 节点明细

机场节点详情中保留路由明细表，字段固定为：

- IP
- 地区
- AS 号
- 组织

规则：

- “地区”只保留 AS 自带的国家或区域结果。
- 不再查询单独的 IP 地理归属信息。

## 成功标准

- 执行后生成一个可直接打开的 HTML 文件。
- 页面默认展示完整关系图。
- 同一个 AS 在全图中只出现一次。
- 点击 AS 或机场节点后，页面下方能展示对应详细信息。
- 页面仍可查看机场节点的具体路由表。
